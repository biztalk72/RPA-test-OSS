# RPA-test-OSS

오픈소스 RPA(Robotic Process Automation) 솔루션 벤치마킹 프로젝트

## 프로젝트 목적

다양한 오픈소스 RPA 솔루션들을 비교 분석하고 테스트하여, 각 솔루션의 장단점과 적합한 사용 사례를 파악합니다.

## 벤치마킹 대상 오픈소스 RPA 솔루션

### 1. TagUI
- **설명**: 커맨드라인 기반의 RPA 도구
- **주요 특징**: 
  - 간단한 스크립트 문법
  - 웹, 데스크탑, API 자동화 지원
  - Python, JavaScript 통합 가능

### 2. Robot Framework
- **설명**: Python 기반의 테스트 자동화 프레임워크 (RPA 확장 지원)
- **주요 특징**:
  - 키워드 기반 테스트 작성
  - 다양한 라이브러리 생태계
  - RPA Framework 라이브러리 제공

### 3. OpenRPA
- **설명**: .NET 기반의 오픈소스 RPA 플랫폼
- **주요 특징**:
  - 시각적 워크플로우 디자이너
  - Windows 데스크탽 자동화에 강점
  - 오케스트레이션 기능 제공

### 4. RPA for Python
- **설명**: Python 라이브러리 형태의 간단한 RPA 도구
- **주요 특징**:
  - 설치 및 사용이 간편
  - 자연어 기반 API
  - 웹 자동화 및 OCR 지원

## 테스트 케이스

### 1. Excel 자동화 테스트
각 RPA 솔루션이 네이티브 애플리케이션(Excel)과 상호작용하는 능력을 평가합니다.

**시나리오**:
1. Excel 파일 열기
2. 특정 셀(예: A1)에 숫자 입력
3. 다른 셀(예: B1)에서 해당 값 참조 (예: =A1*2)
4. A1 셀의 값을 변경
5. B1 셀이 자동으로 업데이트되는지 확인
6. 변경된 값을 읽어서 검증

**평가 항목**:
- Excel 애플리케이션 제어 가능 여부
- 셀 읽기/쓰기 정확성
- 수식 계산 자동 반영 확인
- 안정성 및 에러 핸들링

### 2. 통합 워크플로우 테스트 🆕
실제 RPA 시나리오를 모방한 종합 자동화 테스트

**시나리오**:
1. **Native App 자동화**: macOS Notes 앱에 테스트 노트 생성
2. **웹 스크래핑**: quotes.toscrape.com에서 명언 5개 수집
3. **Excel 다중 시트 통합**: 
   - Sheet1: 비즈니스 데이터 + 계산 수식
   - Sheet2: 수집한 웹 데이터
   - Sheet3: 테스트 결과 대시보드

**평가 항목**:
- 다양한 자동화 유형 통합
- 데이터 소스 간 연계
- 복잡한 워크플로우 처리
- 실제 업무 시나리오 시뮬레이션

**실행 결과**: ✅ 모든 테스트 통과 (9.01초)

## 평가 기준

- **사용 편의성**: 학습 곡선, 설치 난이도
- **기능성**: 지원하는 자동화 유형 (웹, 데스크톱, API 등)
- **확장성**: 커스터마이징 및 통합 가능성
- **성능**: 실행 속도, 안정성
- **커뮤니티**: 문서화, 지원 활성도
- **유지보수**: 업데이트 빈도, 이슈 대응
